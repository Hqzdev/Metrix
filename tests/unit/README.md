Tests unit

Эта папка содержит быстрые unit-тесты.

Назначение

Unit-тесты проверяют один модуль или одну функцию без внешней инфраструктуры.

В этом слое не должно быть подключения к PostgreSQL, Redis, Telegram API, Google Calendar API или Microsoft Graph API.

Структура файлов

api — unit-тесты backend-блока packages/api

Правила

Unit-тест должен быть быстрым и детерминированным.
Если тесту нужно время, дату надо фиксировать явно.

Расширение

Добавление unit-теста:

1. выбрать папку по блоку
2. создать файл с суффиксом .test.ts
3. проверить один сценарий
4. не использовать реальные внешние сервисы
