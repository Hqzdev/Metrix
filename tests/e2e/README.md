Tests e2e

Эта папка предназначена для e2e-тестов пользовательских сценариев.

Назначение

E2E-тесты проверяют полный путь пользователя от начала сценария до результата.
Для Smart Booking это бронирование, отмена, перенос, календарная синхронизация и уведомления.

В этом слое не должно быть настоящих платежей и реальных календарных аккаунтов.

Структура файлов

.gitkeep — маркер пустой папки до появления первых e2e-тестов

Правила

Внешние сервисы заменяются mock-адаптерами.
Тест должен проверять итоговое состояние системы, а не только один вызов функции.

Расширение

Добавление e2e-теста:

1. описать сценарий пользователя
2. подготовить mock-адаптеры для внешних сервисов
3. выполнить сценарий шаг за шагом
4. проверить финальное состояние бронирования
