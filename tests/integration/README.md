Tests integration

Эта папка предназначена для интеграционных тестов.

Назначение

Integration-тесты проверяют работу нескольких слоёв вместе: repository, database, queues, auth и adapters.

В этом слое можно использовать PostgreSQL или Redis, если тест явно проверяет интеграцию с ними.

Структура файлов

.gitkeep — маркер пустой папки до появления первых integration-тестов

Правила

Если тест требует DATABASE_URL или REDIS_URL, он должен проверять наличие env.
Если env не задан, тест пропускается.

Тест не должен использовать production-базу, production-Redis или реальные внешние токены.

Расширение

Добавление integration-теста:

1. создать файл с суффиксом .test.ts
2. проверить env перед запуском сценария
3. подготовить тестовые данные
4. выполнить проверку
5. очистить данные после теста
