Error Catalog

Этот документ объясняет основные ошибки системы.

Зачем нужен каталог ошибок

Чтобы backend, bot, web и операторы одинаково понимали, что произошло.

Основные ошибки

BOOKING_CONFLICT

Слот уже занят или бронирование конфликтует с другой бронью.

INVALID_SIGNATURE

HMAC-подпись неправильная.

REPLAY_DETECTED

Запрос с таким X-Request-Id уже был обработан.

RATE_LIMIT_EXCEEDED

Пользователь делает слишком много запросов.

PAYMENT_HOLD_EXPIRED

Временное удержание слота истекло.

PAYMENT_SAGA_FAILED

Платежный flow не завершился и требует recovery.

DLQ_MESSAGE_NOT_FOUND

Оператор пытается replay-ить сообщение, которого нет.

UNAUTHORIZED

Нет прав или подпись не прошла проверку.

FORBIDDEN

Пользователь известен, но действие ему запрещено.

VALIDATION_ERROR

В запросе неправильные данные.

Правило

Новая публичная ошибка должна быть добавлена в этот каталог и в contracts/OpenAPI, если она видна клиентам.
