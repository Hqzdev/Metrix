Queues And Events

Этот документ объясняет события и очереди.

Зачем нужны события

Не все действия нужно выполнять сразу в HTTP-запросе.

Например:

- отправить уведомление;
- обновить аналитику;
- создать отчет;
- синхронизировать календарь.

Для этого сервис публикует событие, а другой сервис обрабатывает его позже.

Redis Streams

Redis Streams используются как event bus.

Примеры streams:

- stream:booking.created;
- stream:booking.cancelled;
- stream:payment.completed;
- stream:notification.send.

Что происходит при ошибке

Если обработчик упал, сообщение остается pending.
Система пробует обработать его снова.
Если попыток слишком много, сообщение уходит в DLQ.

DLQ

DLQ — dead letter queue.
Это место для сообщений, которые не удалось обработать автоматически.

Админ может:

- посмотреть DLQ;
- понять причину;
- replay-ить сообщение;
- увидеть audit action.

Главное правило

ACK делать только после успешной обработки.
Иначе сообщение можно потерять.
