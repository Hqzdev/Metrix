System Overview

Этот документ объясняет систему целиком.

Что делает Metrix

Metrix помогает бронировать офисные ресурсы: переговорки, рабочие места, залы и похожие вещи.

Пользователь может работать через:

- сайт;
- Telegram-бота.

Из чего состоит система

- apps/web — web-приложение.
- apps/bot — Telegram bot runtime.
- PostgreSQL — главная база.
- Redis — очереди, locks, rate limit и временное состояние.

Главный путь пользователя

1. Пользователь открывает Telegram-бота.
2. bot-gateway принимает update.
3. booking-service показывает ресурсы и слоты.
4. Пользователь выбирает слот.
5. booking-service создает бронь или payment-service создает оплату.
6. notification-service отправляет сообщение.
7. analytics-service получает события и обновляет статистику.

Почему сервисов несколько

Так проще понимать ответственность:

- booking-service отвечает за брони;
- payment-service отвечает за оплату;
- calendar-service отвечает за календари;
- admin-service отвечает за админские действия;
- analytics-service отвечает за статистику.

Где хранится правда

PostgreSQL хранит финальное состояние.
Redis помогает быстро обрабатывать временное состояние, но не является главным хранилищем.
