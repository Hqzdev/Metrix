System Overview

О чем этот файл

Этот документ описывает систему на верхнем уровне:

- какие части есть у продукта;
- как они связаны между собой;
- где проходит граница между web, backend, ботом, интеграциями и аналитикой;
- почему выбран модульный монолит, а не микросервисы на старте.

Это не документ про отдельные файлы кода, а про общую форму системы.

Что здесь должно быть описано

- контекст системы;
- акторы: сотрудник, администратор, внешние сервисы;
- основные потоки данных;
- high-level схема компонентов;
- границы ответственности между основными подсистемами;
- нефункциональные ограничения, которые влияют на архитектуру.

Какие файлы и части проекта это покрывает

Этот документ относится к следующим зонам проекта:

```txt
src/app/
src/modules/
src/integrations/
src/queues/
src/realtime/
src/shared/
```

Какие реальные файлы появятся в проекте

На уровне системного обзора ожидаются такие точки входа:

```txt
src/app/layout.tsx
src/app/page.tsx
src/app/(dashboard)/page.tsx
src/server.ts
src/bootstrap.ts
src/shared/config/env.ts
src/shared/config/runtime.ts
```

Зачем нужен этот файл

Чтобы любой разработчик сначала понял форму системы целиком, а уже потом уходил в детали модулей, API и БД.
