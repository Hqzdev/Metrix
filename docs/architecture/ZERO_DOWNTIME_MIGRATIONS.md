Zero Downtime Migrations

Этот документ объясняет безопасные миграции базы.

Проблема

Если поменять таблицу резко, старый код и новый код могут конфликтовать.
Из-за этого deploy может сломать production.

Подход expand/contract

1. Expand — добавить новое поле или таблицу без удаления старого.
2. Deploy — выкатить код, который умеет работать с новым и старым.
3. Backfill — заполнить новые данные.
4. Switch — переключить чтение на новое поле.
5. Contract — удалить старое только после проверки.

Что нельзя делать резко

- удалять колонку, которую еще читает старый код;
- переименовывать колонку без переходного периода;
- добавлять NOT NULL без default/backfill;
- менять enum без проверки старых значений.

Rollback

Rollback должен быть понятен до deploy.
Если rollback невозможен, это нужно явно записать.
