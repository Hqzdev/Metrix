API Versioning

Этот документ объясняет версионирование API.

Зачем нужна версия

Если клиент уже использует API, нельзя ломать его без предупреждения.

Простое правило

Совместимые изменения можно делать внутри текущей версии.
Ломающие изменения требуют новой версии или migration plan.

Совместимые изменения

- добавить необязательное поле;
- добавить новый endpoint;
- расширить enum осторожно, если клиент готов к unknown values.

Ломающие изменения

- удалить поле;
- поменять тип поля;
- сделать optional поле обязательным;
- изменить смысл статуса;
- поменять формат ошибки.

Что делать при breaking change

1. Описать изменение.
2. Добавить новый contract/OpenAPI.
3. Оставить старый путь на переходный период.
4. Записать migration notes.
