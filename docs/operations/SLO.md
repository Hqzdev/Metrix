SLO

Этот документ объясняет цели надежности.

Что такое SLO

SLO — это обещание по качеству работы системы.
Например: сервис должен быть доступен почти всегда, а ответы должны быть быстрыми.

Основные цели

Availability

Core booking flow должен быть доступен большую часть времени.

Latency

Обычные HTTP-запросы должны отвечать быстро.
Для production evidence нужны p95 и p99.

Errors

5xx должны быть редкими.
Рост ошибок должен давать alert.

Queues

Redis stream lag не должен долго расти.
DLQ не должен наполняться без реакции.

Payment recovery

Failed PaymentSaga должна быть видна оператору.

Что еще нужно

Запустить load tests и записать реальные p95/p99/error rate.
