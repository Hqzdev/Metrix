Telegram Bot technology stack

Этот документ описывает, какие технологии используются в Telegram-части проекта и зачем они нужны.

Коротко

Telegram-бот написан на TypeScript и работает как набор Node.js сервисов.
Локально все сервисы запускаются через Docker Compose.
Постоянные данные лежат в PostgreSQL, быстрые временные данные и очереди — в Redis.

Язык и среда

TypeScript

Основной язык кода.
Он похож на JavaScript, но добавляет типы.
Типы помогают заранее увидеть часть ошибок ещё до запуска программы.

Node.js

Среда, в которой запускается TypeScript/JavaScript код на сервере.
Все bot-сервисы после сборки работают как Node.js приложения.

tsx

Инструмент для запуска TypeScript в режиме разработки.
Он нужен, чтобы не собирать проект вручную после каждого изменения.

typescript

Компилятор TypeScript.
Он проверяет код и собирает его в JavaScript для запуска через Node.js.

Хранение данных

PostgreSQL

Основная база данных.
В ней хранятся бронирования, пользователи, календарные подключения, платежные данные и другие постоянные записи.

Prisma

Инструмент для работы с PostgreSQL из TypeScript.
Он описывает структуру базы и даёт удобный способ читать и менять данные из кода.

PgBouncer

Прокладка между сервисами и PostgreSQL.
Она управляет соединениями с базой, чтобы сервисы не создавали слишком большую нагрузку.

Redis

Быстрое хранилище для временных данных.
В проекте Redis используется для состояния пользователя, защиты от повторных Telegram update, rate limit, очередей и событий.

Очереди и фоновые задачи

BullMQ

Библиотека очередей поверх Redis.
Она помогает выполнять задачи не сразу в ответ пользователю, а в фоне.

Например:

- отправить напоминание
- обновить календарь
- подготовить отчёт

Интеграции

Telegram Bot API

Официальный API Telegram для ботов.
Через него bot-gateway получает сообщения, отправляет ответы, редактирует сообщения, показывает кнопки и работает с оплатой.

Telegram Payments

Механизм оплаты внутри Telegram.
Пользователь получает счёт прямо в чате, оплачивает его, а бот получает событие об успешной оплате.

YooKassa provider token

Токен платёжного провайдера для Telegram Payments.
Он нужен payment-service, чтобы выставлять счета.

Google Calendar API

Интеграция с календарём Google.
Пользователь может подключить календарь, чтобы система учитывала или показывала события календаря.

Google OAuth

Способ безопасно подключить Google Calendar.
Пользователь сам разрешает доступ, а система сохраняет подключение.

Инфраструктура

Docker

Упаковывает каждый сервис вместе с тем, что ему нужно для запуска.

Docker Compose

Запускает сразу всю локальную систему: базу, Redis, bot-gateway и остальные сервисы.

Health endpoints

Проверки здоровья сервиса.
Они показывают, жив ли сервис и может ли он работать с нужными зависимостями.

Безопасность

HMAC-подпись внутренних запросов

Сервисы подписывают запросы друг к другу secret-ключами.
Это помогает проверить, что запрос пришёл от доверенного сервиса.

RBAC

Проверка ролей и прав.
Например, статистика /stats доступна только администраторам.

Audit log

Журнал важных действий.
Он нужен, чтобы понимать, кто и что сделал в системе.

Observability

Structured logs

Логи в понятном для машины формате.
Их проще искать, фильтровать и анализировать.

MetricsRegistry

Внутренний сбор метрик.
Метрики помогают видеть, как работает сервис: ошибки, повторные update, состояние health checks и другие показатели.

Основные файлы

- apps/bot/package.json — workspace для всех bot-сервисов и пакетов
- apps/bot/docker-compose.yml — локальный запуск всей bot-системы
- apps/bot/prisma/schema.prisma — схема базы данных bot-приложения
- apps/bot/services — сервисы
- apps/bot/packages — общие пакеты
- apps/bot/services/bot-gateway/src/index.ts — запуск Telegram gateway
- apps/bot/services/bot-gateway/src/bot.ts — основной сценарий Telegram-бота
