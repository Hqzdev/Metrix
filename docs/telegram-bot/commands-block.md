Telegram Bot src/commands

Этот документ описывает блок apps/bot/src/commands: обработчики пользовательских команд и правила их расширения.

Назначение

Блок src/commands реализует обработчики конкретных команд Telegram.
Каждый обработчик получает сообщение и TelegramClient, формирует ответ через messages.ts и keyboards.ts.

В этом слое не должна находиться бизнес-логика, маршрутизация обновлений, работа с сервисами или прямой доступ к хранилищу.

Структура файлов

command-handlers.ts — функции-обработчики для команд /start и /help

Обработчики

Каждый обработчик принимает объект { message, telegram } и не возвращает значение.

handleStartCommand — отправляет приветственное сообщение с главным меню
handleHelpCommand — отправляет список доступных команд с главным меню

Обработчики вызываются из create-bot.ts при получении соответствующей команды в handleMessage.

Зависимости

src/commands использует:

* keyboards.ts — для генерации inline-клавиатур
* messages.ts — для формирования текста ответа
* типы TelegramClient и TelegramMessage

Прямой доступ к BookingService, env или хранилищу запрещён. Все зависимости передаются через входной объект.

Расширение

Добавление нового обработчика команды:

1. создать функцию handleXxxCommand в command-handlers.ts
2. принять { message, telegram } как единственный параметр
3. использовать messages.ts и keyboards.ts для формирования ответа
4. зарегистрировать команду в telegram-client.ts → setMyCommands
5. добавить маршрутизацию в create-bot.ts → handleMessage

Обработчик не должен содержать условий, проверок прав или состояния — только отправку ответа.
